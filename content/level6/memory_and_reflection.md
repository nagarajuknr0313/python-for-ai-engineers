# Memory and Reflection in Agents

## Concept explanation
Memory allows an agent to retain context from prior runs or conversations, while reflection lets it review its own decisions and improve them. Together they make agents more adaptive.

## Real-world AI use case
A coding agent might remember earlier design choices and reflect on whether a prior implementation was too brittle before proposing a new solution.

## Execution flow
1. Store relevant context from a conversation or task.
2. Recall the stored context when the next task arrives.
3. Ask the agent to reflect on whether the decision was strong.
4. Update the memory with the lesson learned.

## Exercises
- Add a simple memory store to the agent workflow.
- Create a reflection note after each task.
- Show how a second task benefits from the earlier memory.

## Interview questions
- Why is memory useful for long-running agent workflows?
- What does reflection add to an agent?
- How can memory introduce bias or stale information?
