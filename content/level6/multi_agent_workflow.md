# Multi-Agent Workflow

## Concept explanation
A multi-agent workflow assigns different responsibilities to separate agents and coordinates them toward a shared objective.

## Real-world AI use case
One agent can gather facts while another drafts a summary and a third evaluates quality.

## Execution flow
1. Define the roles of each agent.
2. Pass intermediate output from one agent to another.
3. Combine the outputs into a final result.
4. Review the outcome for quality.

## Exercises
- Build a workflow with three agents.
- Make one agent critique the output of another.
- Describe how the workflow could be parallelized.

## Interview questions
- Why would you split work across agents?
- What are the risks of agent coordination?
- How would you monitor a multi-agent workflow?
