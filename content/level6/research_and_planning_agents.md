# Research and Planning Agents

## Concept explanation
A research or planning agent turns a user request into a structured plan and gathers the information needed to complete it. It often combines memory, tool use, and multi-step reasoning.

## Real-world AI use case
A research agent can collect evidence about a new product launch, summarize findings, and outline the next steps for a project team.

## Execution flow
1. Receive the user goal.
2. Break the task into steps.
3. Gather context or evidence.
4. Return a structured plan or report.

## Exercises
- Add a reflection step after the plan is generated.
- Expand the agent to include a memory store.
- Improve the planning logic for longer tasks.

## Interview questions
- What makes an agent different from a simple script?
- Why is planning important in multi-step tasks?
- How can memory improve agent behavior over time?
